# UI/UX 改善提案 — 「みんなが楽しんで使える」もじガチャへ

現状のもじガチャは機能としては一通り揃っている(ガチャ・演出・図鑑・色違い・シェア)一方で、
見た目と体験が「テンプレートに機能を載せた」段階にとどまっており、
**触っていて気持ちいい/また開きたくなる** という部分がまだ弱い。

このドキュメントでは、ヒットしているモバイルアプリの UI/UX パターンを参照しながら、
画面ごとの課題と改善案を優先度付きで提案する。

## 3. 画面別の改善案

### 3.1 全体 — デザイントークンの「おもちゃ屋」化

**現状**: `Colors.light.background = '#ffffff'`、アクセントは `#E5484D` がハードコードで散在。

**提案**:

- 背景をスプラッシュ(`app.json` の `#FFF7ED`)と同じ**クリーム色系**に寄せ、アプリ全体を駄菓子屋/おもちゃ屋のトーンで統一する。
- `#E5484D`(赤)・`#000000`(Xボタン)などのハードコードを `theme.ts` に `accent` / `accentText` として集約する。現在9ファイル以上に赤が直書きされている。
- 見出し・数字系テキストに `Fonts.rounded`(丸ゴシック)を全面採用する。詳細画面(`[id].tsx`)では既に使われており、ガチャ画面・コレクション画面だけ素のシステムフォントなのが不揃い。
- レアリティ色(`rarity.ts`)は現状ロジック用として良い出来なので、これを「アプリの3原色」としてUI装飾にも使い回す。

### 3.2 ガチャ画面 (`src/app/index.tsx`)

**a. CTAボタンをDuolingo式の「立体ボタン」に** (優先度: ★★★)

- 下辺に4〜6pxの濃色シャドウ(`borderBottomWidth` or 疑似的な2層View)を付け、押下時に沈む。
- 待機中は2〜3秒周期でわずかに脈動(scale 1.0→1.03)させて「押して!」を伝える。
- 現在の `opacity: 0.8` だけの押下フィードバックを scale スプリングに置き換える。

**b. 「集める目的」を画面に出す** (優先度: ★★★)

- ヘッダー下の「これまでに n 回まわしました」に加えて、**図鑑進捗チップ**(`23 / 71 あつめた`)+ミニ進捗バーを表示する。タップでコレクションタブへ。
- ポケモン図鑑と同じで、「あと何匹」が見えるだけで回す動機が生まれる。データは `useCollectionStore` から既に取れる。

**c. マシンを生きているように** (優先度: ★★)

- 待機中: 中のカプセルがゆっくり浮き沈みする/たまにマシンが小さく揺れる。
- **マシン本体のタップ(またはハンドルのドラッグ)でも回せる**ようにする。Pokémon GOのボール投げ同様、「モノを直接操作した」感が楽しさの核になる。

**d. 結果カードのレアリティ演出** (優先度: ★★★)

- カード背景/枠をレアリティ色で染める(`RarityColors` + 薄いグラデーション)。超レアは金縁+光沢。
- 超レアのときだけ開封前にカプセルが**虹色/金色に発光**する「確定演出」を入れる(モンスト等の定番)。`capsule-reveal.tsx` は既に `superRareAura` を持っているので、開封"前"段階への追加が中心。
- NEW! バッジをカードの外にはみ出す大きなリボンにし、紙吹雪(`confetti.tsx` 実装済み)をNEW時は増量する。

**e. 排出率(提供割合)の明示** (優先度: ★★)

- 画面隅に「排出率」リンク → レアリティごとの割合一覧をモーダル表示。ソシャゲの業界慣習であり、詳細画面には個別排出率が既にあるため一覧化するだけ。

### 3.3 コレクション画面 (`src/app/collection/index.tsx`)

**a. 五十音表らしくする** (優先度: ★★★)

- 各行の左に **行ラベル(あ/か/さ…)** を付ける。`character-hero.tsx` に `ROW_LABELS` が既にあるので流用できる。
- 清音/濁音/半濁音をセクション見出しで区切る。

**b. 未取得マスに「欲しくなる」情報を** (優先度: ★★)

- 未取得でもレアリティの星だけは見せる(「この行に★3が眠っている!」)。ポケモン図鑑の「?」シルエットと同じ心理。
- 未取得マスタップ時も詳細画面の「まだ獲得していません」ページに飛べるようにする(現在は取得済みのみ `Pressable`)。

**c. マイルストーン演出** (優先度: ★★)

- 進捗バーに 25/50/75/100% の目盛りを付け、到達時にその場で紙吹雪+ハプティクス。
- **コンプリート時は賞状(証書)画面**を出して画像シェアできるようにする → SNSでの拡散導線(ツムツム・2048系の定番)。

**d. タブバーのバッジ** (優先度: ★)

- 新規獲得後、コレクションタブに未読ドットを付けて「見に行く」導線を作る。

### 3.4 演出・フィードバック全般

**a. 効果音** (優先度: ★★★)

- `expo-audio` で最小セットを追加: ハンドル回転の「ガリガリ」、カプセル落下の「コトン」、開封の「ポンッ」、超レアの「キラーン」。設定でON/OFF可能に。
- Candy Crush 系の知見では、音の有無で体感の「ジューシーさ」が段違い。ハプティクス実装(`lib/haptics.ts`)と同じ構造で `lib/sounds.ts` を作れば良い。

**b. Reduce Motion 対応** (優先度: ★)

- 演出を増やす以上、`useReducedMotion`(reanimated)で演出短縮パスを用意しておく。

### 3.5 リテンション(また開きたくなる仕掛け)

**a. デイリーボーナス「今日のラッキー文字」** (優先度: ★★)

- 毎日1文字がピックアップされ、その文字の排出率がアップ+獲得時に特別演出。
- ガチャ画面に「きょうのラッキー文字: ぬ」と表示するだけで、毎日開く理由になる(Duolingoのデイリークエスト相当)。`drawCharacter` は重み注入式なので実装コストが低い。

**b. 連続記録(ストリーク)** (優先度: ★)

- 「n日連続でまわした」カウンタ。`collection.ts` ストアに最終プレイ日を持たせるだけで実現できる。

### 3.6 将来アイデア(P2 / 別Issue化推奨)

- **ことばづくり**: 集めた文字だけで単語を作って飾る/シェアする。コレクションに「使い道」が生まれ、最強のリテンション装置になる。
- 称号・実績(「あ行コンプ」「濁音マスター」)。
- カタカナ・アルファベット等の新弾(シーズン制)。`LanguageSet` 型が既に多言語前提の設計。

## 4. 実装ロードマップ(提案)

| フェーズ | 内容 | 規模感 |
|---------|------|--------|
| **P0: 見た目の即効薬** | 立体CTA+バウンス / 結果カードのレアリティ色 / ガチャ画面に図鑑進捗チップ / 行ラベル / タブ選択色をアクセント化 / 赤色のトークン集約 | 各 小(1PRずつ可) |
| **P1: 楽しさの核** | 効果音 / マシン待機アニメ+直接タップ / 超レア確定演出 / マイルストーン&コンプ演出 / 排出率モーダル | 各 中 |
| **P2: リテンション** | ラッキー文字 / ストリーク / 賞状シェア / ことばづくり | 中〜大 |

各項目は独立して着手できるよう意図的に分割してある。P0 だけでも「テンプレ感」はかなり解消される見込み。

## 5. 効果の確かめかた

- 定性: 実機で子ども/友人に触ってもらい「もう1回まわしたくなるか」を観察(演出スキップしたくなったら長すぎのサイン)。
- 定量(将来): 起動あたりのガチャ回数、翌日再訪率、シェアボタン押下率。
